CU-03: Actualizar Cliente

UMLUseCase
Casos de uso :: Gestión clientes :: CU-03: Actualizar Cliente

Description

Escenarios: CU-03: Actualizar Cliente

Actualizar Cliente (Flujo básico)

1- El caso de uso se inicia cuando Admin selecciona Actualizar cliente

2- El sistema despliega una pantalla con los siguientes puntos - Ingresar identificación del cliente - Salir

3- El administrador ingresa la identificación del cliente

4- El administrador selecciona la opción de actualizar cliente

5- El sistema despliega una pantalla con los siguientes puntos -identificación (recuperar sistema) -Nombre (Recuperar Sistema) -Aperllido (Recuperar Sistema) -Teléfono (Recuperar Sistema) -Dirección (Recuperar Sistema) -Correo (Recuperar Sistema) -Estado (Recuperar Sistema) -Nueva Dirección -Nuevo Correo -Nuevo Teléfono -Nuevo Estado

6- El administrador ingresa datos a los campos que desea actualizar

7- El administrador puede dar clic en actualizar o en cancelar si lo desea

8- El caso de uso finaliza

Flijos alternos

2.1- Si el administrador selecciona la opcion ingresar identificación del cliente

2.1.1- Se continúa al paso 3

2.2- Si el adminstrador selecciona la opción salir

2.2.1- Se finalizará el caso de uso

3.1- Si el administrador ingresa un número de identificación correcto

3.1.1- Se continúa al paso 4

4.1- Si el administrador selecciona actualizar cliente

4.1.1- Se continúa con el paso 5

4.2- Si el administrador decide cancelar la operación

4.2.1- se finaliza el caso de uso

6.2- Si el administrador da clic en nuevo estado se le presentan solo dos opciones a escoger -Activo -Inactivo

6.2.1- El administrador elije una de las opciones

6.2.2- Se continúa con el paso 7

7.1- Si el administrador da clic en actualizar

7.1.1- El sistema despliega un mensaje informando que los datos han sido actualizados con éxito

7.2- Si el administrador da clic en cancelar

7.2.1- El sistema despliega una pantalla con las opciones: -Cancelar operación -Continuar actualizando

7.2.1.1- Si el administrador da clic en cancelar operacion

7.2.1.1.1- se finaliza el caso de uso

7.2.1.2- Si el administrador da clic en continuar actualizando

7.2.1.2.1- Se vuelve al paso 6

Excepciones

3.1- Si el administrador ingresa un número de identificación erroneo

3.1.1- El sistema despliega un mensaje dijiciendo que el número ingresado no se encuentra registrado en la base de datos

3.1.2- Se vuelve al paso 3

3.3- Si el administrador ingresa caracteres erróneos como letras

3.3.1- El sistema despliega un anuncio diciendo que los caracteres ingresados no son válidos

3.3.2- Se vuelve al paso 3

6.1- Si el administrador ingresa datos de carácteres inválidos en la casilla teléfono

6.1.1- El sistema despliega un mensaje informando que los caracteres ingresados en la casilla teléfono son inválidos

6.1.2- Se vuelve al paso 6

6.2.1- Si el administrador señala activo y el estado actual ya lo es

6.2.1.1- El sistema despliega un mensaje informando que el cliente se encuentra activo actualmente

6.2.1.1.1- Se vuelve al paso 6

6.2.2- Si el administrador señala inactivo y el estado actual ya lo es

6.2.2.1- El sistema despliega un mensaje informando que el cliente se encuentra inactivo actualmente

6.2.2.1.1- Se vuelve al paso 6
